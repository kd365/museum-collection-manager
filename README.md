# Museum Collection Manager

A full-stack web application for managing museum collections, artworks, artists, and exhibitions. Built with Flask and SQLAlchemy, featuring AI-powered artwork and collection descriptions generated by AWS Bedrock.

**Source Code:** [https://github.com/kd365/museum-collection-manager](https://github.com/kd365/museum-collection-manager)

---

## Overview

Originally built as a Flask + MySQL CRUD application, this project was modernized with a focus on production-readiness and AI integration. The current version replaces raw SQL with an ORM, adds role-based authentication, fixes a SQL injection vulnerability, and introduces AI features that generate curator-style artwork descriptions and collection overviews using AWS Bedrock.

## Key Features

- **Artist Management**: Track artist biographies, birth/death dates, art movements, nationalities, and portfolios
- **Artwork Cataloging**: Detailed records with dimensions, valuations, signatures, provenance, and AI-generated descriptions
- **Museum Registry**: Museum information including locations, visitor statistics, admission fees, and AI-generated collection overviews
- **Collection Management**: Link artworks to museums with acquisition details, accession numbers, and display status
- **AI Artwork Descriptions**: Generate museum placard-style descriptions for any artwork using Claude Sonnet 4 on AWS Bedrock
- **AI Collection Overviews**: Automatically generates a collection summary when a museum accumulates 5+ artworks
- **SQL Playground**: Secured interactive query builder with parameterized queries and column dropdowns
- **Role-Based Access**: Admin users can create/edit/delete; visitors can browse, search, and export
- **Pagination**: All list pages paginated for performance
- **Data Export**: Export any table to CSV

## Modernization: What Changed

This project went through a significant refactor. Here's what was improved and why:

| Before | After | Why |
|--------|-------|-----|
| Raw MySQL queries (20+ duplicated connection blocks) | SQLAlchemy ORM with models | Eliminates code duplication, enables relationships, connection pooling |
| No authentication | Flask-Login with admin/viewer roles | Protects data integrity on public deployments |
| SQL injection in Playground (raw WHERE clause) | Parameterized queries + column dropdown | Fixes a textbook OWASP vulnerability |
| `print()` error handling | Python `logging` module | Structured logging for debugging and monitoring |
| MySQL required | SQLite for dev, MySQL for production | Zero-setup local development |
| No containerization | Docker + docker-compose | One-command deployment |
| No AI features | Bedrock-powered descriptions | Demonstrates AI/ML integration |

## AI Integration

The app uses AWS Bedrock (Claude Sonnet 4) to generate two types of descriptions:

**Artwork Descriptions** — When an admin clicks "Generate AI Description" on an artwork, the system builds a prompt from the artwork's metadata (title, artist, medium, movement, dimensions, value) and generates a 2-3 sentence museum placard description.

**Collection Overviews** — When a museum's collection reaches 5+ artworks, the system automatically generates a thematic overview describing the collection's artistic movements, notable pieces, and curatorial narrative. This triggers once and is stored with the museum record.

Both features use the `cyber-risk` AWS profile and call Bedrock via `boto3` with structured prompts that specify the curator persona and output constraints.

| Authentication | Flask-Login with role-based access |

| Containerization | Docker + docker-compose |
| Production | Gunicorn WSGI server |

## Security

- **SQL Injection Fixed**: The SQL Playground previously interpolated raw user input into queries. Now uses parameterized queries with a column dropdown that validates against actual table schemas.
- **Role-Based Access**: Admin-only routes for create/edit/delete. Visitors can browse, use SQL Playground (read-only), and export CSVs.
- **CSRF Protection**: All forms protected via Flask-WTF tokens.
- **Secure Sessions**: HTTPOnly, SameSite, and Secure cookie flags in production.
- **Environment Secrets**: All credentials via `.env` file, never committed.

## Getting Started

```bash
git clone https://github.com/kd365/museum-collection-manager.git
cd museum-collection-manager
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
python seed_data.py    # Pre-populate with sample data
python museums_app.py  # Starts on http://localhost:5001
```

**Admin login:** `admin@museumcollection.com` / `Admin123!`

### Docker

```bash
docker compose up
```
